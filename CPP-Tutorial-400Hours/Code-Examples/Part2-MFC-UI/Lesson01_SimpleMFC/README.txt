================================================================================
LESSON 01: SIMPLE MFC APPLICATION
================================================================================

WHAT THIS DEMONSTRATES:
- Basic MFC dialog application structure
- CWinApp and CDialog classes
- Message maps and event handling
- DDX (Dialog Data Exchange)
- Simple button click handler

================================================================================
FILE STRUCTURE:
================================================================================

StdAfx.h / StdAfx.cpp
    - Precompiled header files
    - Includes all MFC headers
    - Speeds up compilation

SimpleMFCApp.h / SimpleMFCApp.cpp
    - Application class (CWinApp derived)
    - Contains InitInstance() - application entry point
    - Creates and shows the main dialog

SimpleMFCDlg.h / SimpleMFCDlg.cpp
    - Main dialog class (CDialogEx derived)
    - Handles user interface
    - Contains message handlers (OnButtonHello)
    - Implements DDX for data exchange

Resource.h
    - Resource ID definitions
    - Generated by Visual Studio

SimpleMFC.rc
    - Resource file containing dialog layout
    - Defines controls, positions, and properties

================================================================================
HOW TO COMPILE IN VISUAL STUDIO:
================================================================================

1. Create new MFC Application project (Dialog-based)
2. Replace generated files with these files
3. Add SimpleMFC.ico to res\ folder (or create a simple icon)
4. Build -> Build Solution (Ctrl+Shift+B)
5. Run (F5)

OR manually:
1. File -> New -> Project
2. Select "MFC Application"
3. Choose "Dialog based"
4. Copy these .cpp and .h files over the generated ones
5. Build and run

================================================================================
KEY MFC CONCEPTS:
================================================================================

1. APPLICATION OBJECT (theApp)
   - Every MFC app has ONE CWinApp-derived object
   - Controls application lifecycle
   - InitInstance() is called at startup

2. DIALOG CLASS
   - Derived from CDialog or CDialogEx
   - Represents the main window
   - Handles user interactions

3. MESSAGE MAPS
   - Connect Windows messages to C++ functions
   - BEGIN_MESSAGE_MAP / END_MESSAGE_MAP macros
   - ON_BN_CLICKED connects button clicks to handlers

4. DDX (Dialog Data Exchange)
   - Automatic data transfer between variables and controls
   - DoDataExchange() function
   - UpdateData(TRUE) = control -> variable
   - UpdateData(FALSE) = variable -> control

5. RESOURCE FILE (.rc)
   - Defines dialog layout visually
   - Contains control IDs, positions, sizes
   - Edited with Visual Studio Resource Editor

================================================================================
EXECUTION FLOW:
================================================================================

1. Windows calls WinMain (provided by MFC framework)
2. MFC creates 'theApp' global object
3. WinMain calls theApp.InitInstance()
4. InitInstance creates CSimpleMFCDlg dialog
5. dlg.DoModal() displays dialog and runs message loop
6. User clicks button
7. Windows sends WM_COMMAND message
8. Message map routes to OnButtonHello()
9. OnButtonHello() updates m_strOutput
10. UpdateData(FALSE) displays text in edit control
11. User clicks OK/Cancel
12. DoModal() returns
13. InitInstance returns FALSE
14. Application exits

================================================================================
TO EXTEND THIS APPLICATION:
================================================================================

1. Add more buttons:
   - Add button in .rc file
   - Add ID to Resource.h
   - Add ON_BN_CLICKED to message map
   - Add handler function

2. Add more controls:
   - Edit controls, list boxes, combo boxes
   - Use DDX_Text, DDX_LBString, DDX_CBString

3. Add validation:
   - Use DDV_* functions in DoDataExchange()

4. Add dialogs:
   - Create new dialog resource
   - Create new CDialog-derived class
   - Show with DoModal()

================================================================================
COMMON ISSUES:
================================================================================

1. "Cannot find SimpleMFC.ico"
   - Create a simple icon file or copy from another project
   - Place in res\ subfolder

2. "Unresolved external symbol _main"
   - Project must be configured as MFC application
   - Project Properties -> Use MFC -> Use MFC in a Shared DLL

3. "StdAfx.h not found"
   - Enable precompiled headers in project settings
   - Set StdAfx.cpp to "Create" precompiled header
   - Set other .cpp files to "Use" precompiled header
