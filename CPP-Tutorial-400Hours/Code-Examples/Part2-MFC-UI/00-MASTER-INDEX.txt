HOW TO USE THESE LESSONS
================================================================================

1. STUDY THE CODE:
   - Each lesson folder contains complete, compilable code
   - Read the README.txt in each folder for overview
   - Study the .cpp and .h files for implementation details
   - Comments explain WHERE TO LOOK for key concepts

2. COMPILE IN VISUAL STUDIO:
   For each lesson:
   a) Create new MFC Application project in Visual Studio
   b) Choose appropriate type (Dialog-based or Doc/View)
   c) Copy files from lesson folder to your project
   d) Replace generated files with lesson files
   e) Build Solution (Ctrl+Shift+B)
   f) Run (F5)

3. MODIFY AND EXPERIMENT:
   - Each lesson is a starting point
   - Add your own features
   - Experiment with variations
   - Break it and fix it to learn

4. PROGRESSIVE LEARNING:
   - Follow lessons in order for best results
   - Lessons 01-15: Master dialogs first
   - Lessons 16-30: Understand Doc/View architecture
   - Lessons 31-45: Learn all controls
   - Lessons 46-60: Master GDI graphics
   - Lessons 61-75: Advanced topics
   - Lessons 76-90: Build complete applications

================================================================================
PROJECT SETTINGS FOR VISUAL STUDIO
================================================================================

REQUIRED SETTINGS:
1. Configuration Properties -> General
   - Use MFC: Use MFC in a Shared DLL (or Static Library)
   - Character Set: Use Unicode Character Set

2. C/C++ -> Precompiled Headers
   - Precompiled Header: Use (/Yu)
   - Precompiled Header File: StdAfx.h
   - For StdAfx.cpp: Create (/Yc)

3. C/C++ -> General
   - Warning Level: Level 3 or 4
   - Treat Warnings as Errors: No

4. Linker -> System
   - SubSystem: Windows (/SUBSYSTEM:WINDOWS)

================================================================================
TROUBLESHOOTING
================================================================================

COMMON ISSUES:

1. "Cannot find StdAfx.h"
   Solution: Enable precompiled headers in project settings

2. "Unresolved external symbol _main"
   Solution: Set SubSystem to Windows, not Console

3. "MFC classes not found"
   Solution: Change "Use MFC" to "Use MFC in a Shared DLL"

4. "Resource file errors"
   Solution: Ensure .rc file is included in project

5. "Unicode conversion errors"
   Solution: Use _T() macro for string literals

6. "Missing .ico files"
   Solution: Create simple icon files or copy from another project
