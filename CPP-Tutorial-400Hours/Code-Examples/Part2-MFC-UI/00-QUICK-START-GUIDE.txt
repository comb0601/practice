CREATING A NEW PROJECT FOR EACH LESSON
--------------------------------------

1. START VISUAL STUDIO
   File -> New -> Project

2. SELECT MFC APPLICATION
   - Filter: C++, Windows, Desktop
   - Template: "MFC Application"
   - Name: Lesson{number}_{name} (e.g., Lesson01_SimpleMFC)
   - Location: Your workspace folder
   - Click "Create"

3. MFC APPLICATION WIZARD

   PAGE 1 - Application Type:
   For Lessons 01-15, 31-45, 46-60, 61-75, 76-90:
     • Application type: Dialog based
     • Visual style: Windows Native/Default
     • Use Unicode libraries: Yes

   For Lessons 16-30 (Doc/View):
     • Application type: Single document (SDI)
     • Project style: MFC Standard
     • Document/View architecture support: Yes
     • Use Unicode libraries: Yes

   PAGE 2 - User Interface Features:
     • Thick frame: Yes
     • Minimize/Maximize boxes: Yes
     • System menu: Yes
     • About box: Yes (optional)
     • Initial status bar: Yes (for Doc/View)
     • Initial toolbar: Yes (for Doc/View)

   PAGE 3 - Advanced Features:
     • Use default settings
     • Windows Sockets: Optional (needed for Lesson 70, 82)

   PAGE 4 - Generated Classes:
     • Use default class names or customize
     • Click "Finish"

4. COPY LESSON FILES
   - Wait for wizard to generate project
   - Close Visual Studio temporarily
   - Navigate to lesson folder in File Explorer
   - Copy ALL files (.h, .cpp, .rc, Resource.h, StdAfx.h, etc.)
   - Paste into your project folder
   - Overwrite existing files

5. REFRESH PROJECT IN VISUAL STUDIO
   - Reopen Visual Studio
   - If prompted about changed files, click "Reload All"
   - Solution Explorer should show all lesson files

6. CONFIGURE BUILD SETTINGS
   Right-click project -> Properties:

   Configuration: All Configurations
   Platform: All Platforms

   Configuration Properties -> General:
     • Use of MFC: Use MFC in a Shared DLL
     • Character Set: Use Unicode Character Set

   C/C++ -> Precompiled Headers:
     • Precompiled Header: Use (/Yu)
     • Precompiled Header File: StdAfx.h

   For StdAfx.cpp only:
     • Right-click StdAfx.cpp -> Properties
     • C/C++ -> Precompiled Headers
     • Precompiled Header: Create (/Yc)

7. BUILD
   • Build -> Build Solution (Ctrl+Shift+B)
   • Check Output window for errors
   • If successful: "Build succeeded"

8. RUN
   • Debug -> Start Debugging (F5)
   • Application window appears
   • Test the functionality

================================================================================
ALTERNATIVE: MANUAL PROJECT SETUP (Advanced)
================================================================================

If you prefer to manually set up without wizard:

1. CREATE EMPTY MFC PROJECT
   - New Project -> Empty Project
   - Add MFC support in project properties

2. ADD LESSON FILES
   - Right-click project -> Add -> Existing Item
   - Select ALL .cpp files from lesson folder
   - Repeat for .h files
   - Add .rc file (Resource Script)

3. ADD RESOURCE FILE
   - Right-click project -> Add -> Resource
   - Import .rc file from lesson folder

4. CONFIGURE INCLUDES
   - Project -> Properties
   - C/C++ -> General -> Additional Include Directories
   - Add: $(ProjectDir)

5. SET ENTRY POINT
   - Linker -> System
   - SubSystem: Windows (/SUBSYSTEM:WINDOWS)

6. BUILD AND RUN
   - Build Solution
   - Start Debugging

COMMON BUILD ERRORS AND SOLUTIONS
================================================================================

ERROR: "Cannot open include file 'StdAfx.h'"
SOLUTION:
  1. Verify StdAfx.h exists in project folder
  2. Check precompiled header settings
  3. Ensure StdAfx.h is first include in all .cpp files

ERROR: "Unresolved external symbol _main"
SOLUTION:
  1. Project Properties -> Linker -> System
  2. SubSystem: Windows (not Console)
  3. Rebuild project

ERROR: "'CWinApp': identifier not found"
SOLUTION:
  1. Project Properties -> Configuration Properties -> General
  2. Use of MFC: Use MFC in a Shared DLL
  3. Clean and rebuild

ERROR: "'_TCHAR' undeclared identifier"
SOLUTION:
  1. Include windows.h or afxwin.h
  2. Or use TCHAR instead of _TCHAR

ERROR: "Resource file errors"
SOLUTION:
  1. Ensure .rc file is added to project
  2. Resource.h must match .rc resource IDs
  3. Check for duplicate resource IDs

ERROR: "LNK2019: unresolved external symbol"
SOLUTION:
  1. Verify all .cpp files are included in project
  2. Check that functions are properly defined
  3. Ensure proper linkage (extern "C" if needed)

ERROR: "Cannot find resource file"
SOLUTION:
  1. Create res\ subfolder in project directory
  2. Add placeholder .ico files
  3. Or remove icon references from .rc file

ERROR: "Unicode conversion error"
SOLUTION:
  1. Use _T() macro: _T("string")
  2. Or use L prefix: L"string"
  3. Or disable Unicode: Project Properties -> Character Set: Not Set
